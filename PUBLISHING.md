# Publishing 007 First Light to the Connect IQ Store

This is the end-to-end path from this repo to a live listing. Everything that
can be automated here is; the final upload is a manual, authenticated step in
Garmin's developer dashboard that only you can perform (it needs your Garmin
account and your signing key).

> TL;DR: get a developer key → build the signed `.iq` (CI/release does this) →
> upload it at the Connect IQ developer dashboard with the listing copy and real
> screenshots → submit for review.

---

## 1. One-time prerequisites

1. **Garmin account** with Connect IQ developer access — sign in at
   <https://apps.garmin.com/> and open the developer dashboard
   (<https://apps.garmin.com/developer/dashboard>). Accept the developer agreement.
2. **A developer signing key.** This key *is your identity* for every app you
   publish and every future update — generate it once and keep it safe:

   ```bash
   openssl genrsa -out developer_key.pem 4096
   openssl pkcs8 -topk8 -inform PEM -outform DER -nocrypt \
     -in developer_key.pem -out developer_key.der
   ```

   Store `developer_key.der` somewhere durable (a password manager / secure
   backup). If you lose it you cannot push updates to the same store entry. It is
   git-ignored here on purpose — never commit it.

3. *(Optional, for automated builds)* add the key as a CI secret so the
   `package`/`release` workflows sign the real package:

   ```bash
   base64 -w0 developer_key.der   # copy the output
   ```

   GitHub → repo **Settings → Secrets and variables → Actions → New secret**,
   name `CIQ_DEVELOPER_KEY_B64`, paste the base64. Without it, CI still builds a
   `.iq` but signs it with a throwaway key (fine to inspect, not for submission).

## 2. Build the store package (`.iq`)

Pick one:

- **From CI (recommended):** every push builds `007FirstLight-iq` as a workflow
  artifact. With `CIQ_DEVELOPER_KEY_B64` set it is correctly signed. Download it
  from the run's Artifacts section.
- **From a release:** tag a version and the release workflow attaches the `.iq`
  to a GitHub Release:

  ```bash
  git tag v1.0.0 && git push origin v1.0.0
  ```

- **Locally** (needs the Connect IQ SDK installed):

  ```bash
  monkeyc -e -f monkey.jungle -o dist/007FirstLight.iq -y developer_key.der
  ```

The `-e` (export) build packages every product listed in `manifest.xml`.

## 3. Capture screenshots (required by the store)

Garmin wants real screenshots at the device resolution (454×454 for the Fenix 8
Pro 47mm), not renders. In the Connect IQ **simulator**:

1. `connectiq` (start the simulator), then
   `monkeydo bin/007FirstLight.prg fenix8pro47mm`.
2. Set an interesting time (e.g. 10:09) and toggle a couple of settings via
   **Settings → Edit Persistent Storage / App Settings**.
3. Capture with **File → Capture Simulator Screen** (saves a PNG). Grab a few:
   default look, the Dawn theme, an always-on frame.

The mockups in [`store/screenshots/`](store/screenshots/) (generated by
`python3 tools/gen_preview.py`) are good for the README and as placeholders, but
upload real simulator captures for the actual submission.

## 4. Create the store entry

In the developer dashboard, **Upload an App**:

1. Upload `007FirstLight.iq`. The dashboard reads the app UUID, type (watch
   face) and compatible devices from the package.
2. Fill the listing from [`store/listing.md`](store/listing.md): name, short and
   long descriptions, category (Watch Faces), keywords, languages, price (Free).
3. Add the store icon (use `resources/drawables/launcher_icon.png` or a larger
   render) and the screenshots from step 3.
4. Review the permissions screen — explain **SensorHistory** using
   [`store/PERMISSIONS.md`](store/PERMISSIONS.md).
5. Set the support URL and, since the face only reads sensor data locally, note
   that no data leaves the device (no privacy policy URL needed for network use,
   because there is none).
6. Set the version label (e.g. `1.0.0`) and the "What's new" text.

## 5. Submit for review

Submit the app. Garmin runs an automated + manual review (typically a few days).
Watch for review feedback in the dashboard / your account email and resubmit if
they request changes.

## 6. Shipping updates later

1. Make changes on a branch; let CI compile + package it green.
2. Bump the version label and update "What's new".
3. Build a new `.iq` **with the same `developer_key.der` and the same app UUID**
   (the UUID lives in `manifest.xml` — do not change it) and upload it as a new
   version to the existing store entry.

---

## Notes & caveats

- **Watch faces are not affected** by the Fenix 8 Pro's LTE rule that restricts
  *app* installs to Garmin-certified titles — faces sideload and distribute
  through the store normally.
- This is an unofficial tribute. Before publishing publicly, make sure your use
  of "Omega", "Seamaster" and "007" naming/branding fits Garmin's store policy
  and trademark rules — you may need to soften the name/description.
- Keep `developer_key.der` and the `CIQ_DEVELOPER_KEY_B64` secret private.
